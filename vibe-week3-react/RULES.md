# RULES.md

## Development Rules

1. Keep the project beginner-friendly.
2. Use TypeScript as part of the React practice.
3. Keep types simple and practical.
4. Do not add complex folder structures too early.
5. Do not install extra packages unless necessary.
6. Start from `src/App.tsx` and `src/App.css`.
7. Make one React concept clear at a time.
8. Do not mix unrelated features in one change.
9. Always keep the app runnable.
10. Prefer readable code over clever code.
11. Preserve working code.

## TypeScript Rules

- Use simple types when they clarify props, state, or data.
- Avoid advanced generics in beginner lessons.
- Avoid complex utility types unless explicitly needed.
- Do not convert the project away from TypeScript.

## Project Boundary Rules

- Work inside `vibe-week3-react`.
- Do not modify `../what-is-eat`.
- Do not modify `../what-is-score`.
- Do not modify root workspace documents unless the user explicitly approves it.

## Git Rules

- Do not auto commit.
- Do not auto push.
- Suggest a commit message only when the user asks or after meaningful changes.

## Verification Rules

- Run `npm run build` after code changes.
- Run `npm run lint` when checking code quality.
- If a command fails, explain the failure and the smallest next fix.
